## Этап 3 — Лог использования ✅

> **Статус:** Готово
> **Кто использует:** Курьер/Супервайзер

- [x] **BIKE-30** Хэндлер «Взял байк» — курьер отмечает начало смены (выбор байка по складу)
- [x] **BIKE-31** Хэндлер «Вернул байк» — курьер отмечает конец смены
- [x] **BIKE-32** Автозакрытие лога — если курьер забыл нажать «Вернул», автоматически закрывать через N часов
- [x] **BIKE-33** Хэндлер «Кто сейчас на байке» — показать активные смены по складу

## Этап 4 — Фиксация поломок ✅

> **Статус:** Готово
> **Кто использует:** Супервайзер

- [x] **BIKE-40** Хэндлер «Создать поломку» — FSM (выбор байка → тип поломки → описание)

- [x] **BIKE-44** Хэндлер «История поломок байка» — показать все поломки конкретного байка

## Этап 5 — Процесс ремонта ✅

> **Статус:** Готово
> **Кто использует:** Мастер

- [x] **BIKE-50** Хэндлер «Забрал байк» — мастер нажимает, фиксируется `picked_up_at`, статус байка → `repair`
- [x] **BIKE-51** Хэндлер «Байк готов» — мастер заполняет: что сделано, время, стоимость. Статус байка → `online`
- [x] **BIKE-52** Хэндлер «Мои ремонты» — мастер видит свои текущие и завершённые ремонты
- [x] **BIKE-53** Связь ремонта с поломкой — при создании ремонта привязать к карточке поломки

## Этап 7 — Аналитика ✅

> **Статус:** Готово
> **Кто использует:** Админ

### Поломки
- [x] **BIKE-70** Отчёт «Поломки за месяц» — общее кол-во, по складам, по типам
- [x] **BIKE-71** Отчёт «Поломки по курьерам» — рейтинг: кто чаще ломает

### Байки
- [x] **BIKE-72** Отчёт «Самые ненадёжные байки» — топ байков по количеству поломок
- [x] **BIKE-73** Отчёт «Ремонты байка» — сколько ремонтов, сколько дней в ремонте
- [x] **BIKE-74** Отчёт **Downtime** — `(часы в ремонте / общее время) × 100%` по каждому байку

### Курьеры
- [x] **BIKE-75** Отчёт «Аккуратные курьеры» — рейтинг курьеров (кто не ломает)
